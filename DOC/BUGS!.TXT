#include <stdio.h>

char buffer[4]="AAA";
//char buffer[4]={'A','A','A','A'};

void main()
{
    printf("0x%02X\n", *buffer);		// (1) Not correct!
    printf("0x%02X\n", (char *) *buffer);	// (2) Correct
}

If the buffer array is represented as a char array, then the result obtained as shown
in line (1), will be generated incorrectly until such time as it is needed.
Moreover, the compiler will not report an error. In order to generate the correct
value, and in a separate case with a char array, you must register it the way it is
done in line (2), necessarily specifying the required cast to tell the compiler what
you need.
When working with arrays represented as int elements, there are obvious problems
with proper execution, generating correct results.
To avoid such problems with the representation, and what's more, in order to view asm
code generated by the compiler for the given example, for line (1) the generated
instruction is "ld b,0".
